# `book_config.json` schema

Tells `build_epub.py` how to clean up and structure *this specific book*. The assembled OCR markdown alone doesn't have enough signal to distinguish a real chapter marker from a page header, or a front-matter section title from its body — this file encodes those judgments.

See `sample/hilbert-book-config.json` for a complete worked example.

## Metadata

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `title` | string | — | Book title (goes into EPUB metadata + as H1) |
| `author` | string | — | Goes into EPUB metadata |
| `translator` | string | — | EPUB metadata only |
| `language` | string | `zh-CN` | pandoc `--metadata lang=...` |
| `author_line` | string | — | Subtitle printed below the H1 (e.g. "Author 著；Translator 译") |

## Input/output paths

| Field | Default | Notes |
|-------|---------|-------|
| `source_md` | `hilbert.md` | Input: concatenated OCR markdown in `$WORK_DIR/` |
| `output_md` | `book_clean.md` | Intermediate clean markdown written alongside source |

## Chapter detection

| Field | Default | Notes |
|-------|---------|-------|
| `chapter_pattern` | `^第([一二三四五六七八九十廿卅百0-9]+)章[\\s·•]*(.*)$` | Regex with 2 capture groups: (1) chapter number, (2) inline title. The pattern must only match lines that *could* be a chapter marker — running `re.match` against every line is how candidates are collected. |
| `chapter_count` | `99` | Maximum valid chapter number. A chapter candidate parsing to a larger number is rejected (filters noise like "第一一十五章"). |
| `chapter_titles` | `{}` | Map of chapter number (stringified int) → title. When set, overrides any title captured inline. Generate this from the TOC page — the inline titles are usually missing/garbled because OCR loses the space between `第N章` and the title. |

The detector distinguishes TOC entries from real chapter starts: if two chapter candidates with **different** numbers appear within 20 non-blank lines, both are treated as TOC and skipped. Page-header repeats (same number appearing many times) are also collapsed — only the first occurrence of each number becomes a real heading.

## Noise filtering

| Field | Default | Notes |
|-------|---------|-------|
| `running_headers` | `[]` | Exact strings that appear alone on a line and should be dropped. Use for book-title running headers ("希尔伯特") and chapter-name running headers (e.g. "青年时代" if it recurs). |
| `orphan_fragments` | `["译注", "——译注", "—译注", "-—译注"]` | Lone lines that are footnote-terminator scraps without the footnote body. |
| `skip_exact_lines` | `[]` | Other exact lines to skip (e.g. the OCR script's subtitle line). |
| `book_title_heading_prefix` | `""` | If the OCR wrote the book title as an H1 at the top, set this prefix (e.g. `"# 希尔伯特"`) so those lines get dropped and we emit our own H1 from the `title` field. |

Single CJK characters alone on a line are **always** dropped (they're almost certainly side-margin vertical text). Bare 1–3 digit numbers are always dropped (page numbers).

## Image plates

| Field | Default | Notes |
|-------|---------|-------|
| `image_plate_range` | `null` | `[lo, hi]` page range where low-text pages should be embedded as full-page JPEG images. Leave null to skip image embedding. |
| `image_plate_text_threshold` | `50` | A page within the plate range is treated as image-only if it has < this many CJK+Latin characters. |

JPEGs are generated via `magick <png> -resize 1000x -quality 80` from the rendered 200-DPI PNGs.

## Trailers

| Field | Default | Notes |
|-------|---------|-------|
| `trailer_markers` | `["Document generated by Anna"]` | Strings that signal the start of PDF-tool provenance metadata at the end of the book. The script trims from the nearest `<!-- page N -->` before each marker. |

## Heading promotions (front/back-matter)

`heading_promotions` is a list of rules. Each rule can have:

```json
{
  "pattern": "regex",
  "title": "section title",
  "replacement": "optional — overrides the default `\n\n## <title>\n\n`",
  "flags": ["MULTILINE", "DOTALL"]
}
```

Typical use: detect run-together section titles like `出版说明自中西文明...` and promote them to proper `##` headings. The regex usually needs a positive lookahead to anchor on the first few characters of the section body, because the raw section title string might appear elsewhere in the book.

Examples from the Hilbert config:

```json
{"pattern": "^出版说明(?=自中西文明)", "title": "出版说明", "flags": ["MULTILINE"]}
{"pattern": "(内容简介)[:：]\\s*", "title": "内容简介"}
{"pattern": "(\\n\\s*)后记\\s*(?=“大卫·希尔伯特是他那个时代真正伟大)",
 "title": "后记",
 "replacement": "\\1## 后记\n\n"}
```

## Drop patterns

`drop_patterns`: list of regex strings. Any match (with MULTILINE flag) is removed. Use for garbled TOC residue that survived other filters.

## Pandoc options

| Field | Default | Notes |
|-------|---------|-------|
| `toc_depth` | `2` | `--toc-depth` |
| `split_level` | `2` | `--split-level` — level 2 means one XHTML file per `##` heading |

## Tips for generating a good config

- **Chapter titles are the single most important field.** If they're wrong, the EPUB's navigation is wrong. Invest time reading the TOC page carefully.
- **Start minimal**, rebuild, inspect the output, add fields as issues appear. You don't need to populate every field upfront.
- **`heading_promotions` patterns should be as specific as possible** (long lookaheads with unique body text) to avoid false positives. A rule that fires in the wrong place can slice a chapter in two.
- **Use `sample/hilbert-book-config.json`** as a starting point when the book is Chinese — the regex patterns for 第N章 and common front-matter titles apply broadly.
